The Story

My personal site has been 'in development' for a long time. My web development path has been circuitous and disjointed, much like many of my peers, and I was always told that having a personal website was a really good way to get work in front of people.

It started as a single hand-coded HTML page with a short stylesheet and a link to my pdf resume. Take a look at it here.

I got attached to this basic layout, got precious about it. I wasted a lot of time figuring out how to attach a blog to it without losing the simplicity of its design.

In the end, it was consuming more energy than just shipping the damn thing, so I dove into the world of static site generators to find a solution. Rather than having a database of content that responds to user calls, static site generators do all of the work up front. This leads to sites that are small, fast, and (generally) easy for laypeople to modify.

And the winner is...

After evaluating several different static site generators, Spress had the right combination of traits to entice me into using it:

  • It's open source
  • It's written in PHP, the language I'm using to learn web development
  • It has reasonably up-to-date documentation
  • Generating new content is as easy as writing a markdown file.

This site uses a slightly modified version of the Spresso theme. The design is simple, legible, and easy enough to modify.

I've submitted pull requests to the design's repository and the maintainer is more than willing to work with them. An aside: Getting a PR merged was a huge confidence boost as a developer.

Takeaways

There is always more work to be done, but the main lesson I learned from this project is something that I have learned and relearned multiple times in my adult life:

Just finish it.

Until a work is finished, it may as well not exist. Remove barriers to finishing work and put it in front of people.
